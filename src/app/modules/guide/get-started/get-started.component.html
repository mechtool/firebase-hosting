Использование Firebase Hosting

Для того, что бы использовать возможности хостинга нужно зарегистрироваться на сайте Firebase, перейти в консоль разработчик, создать новый проект проект и совершить необходимые действия, которые подскажет помощник создания проекта.

1. Установить Firebase CLI.

Firebase CLI - оболочка командного интерфейса, для установки которой нужно установить NodeJs, который должет быть не ниже версии 5.10.0 и npm. Эти две программы устанавливаются вместе, в случаи установки NodeJs. Как только установка NodeJs закончится, можно установить библиотеку Firebase CLI, вызвав команду в терминале командной строки операционной системы:

Для правильной работы Firebase CLI требуется установить Node.Js версии не менее 5.10.0

Как только NodeJs и npm установлены, можно установить Firebase CLI через файловый менеджер npm, которая устанавливает иструмент коммандной строки для работы с Firebase,и в пространстве имен глобальных переменных операционной системы появляется переменная firebase 

> npm install -g firebase-tools

2. Получить доступ к проектам Firebase вашего аккаунта.

Чтобы соеденить локальную машину с областью проектов вашего аккаунта Firebase, запустите команду, и введите необходимые данные полномочия вашего аккаунта, которые затребует система авторизации: 

> firebase login

3. Инициализировать приложение.

Для связи проекта на локальной машине с проектом в аккаунте Firebase, необходимо инициализировать корневую папку приложения, для этого нужно в командной оболочке оперпционной системы перейти в папку проекта и вызывать команду инициализации Firebase приложения. Эта команда запустит процесс инициализации, в процессе которого нужно будет ответить на нескольоко вопросов, после которых будет создасдано несколько файлов в корневой папке проекта, среди которых файл конфигурации firebase.json. Более подробно о настройке поведения хостинга. 

> firebase init

4. Определить корневую папку приложения

При выполнении команды инициализации будет предложено указать на корневую папку приложения, т.е. такую папку, в которой будут находиться готовые для развертывания, аглифицированные и модифицированные файлы непосредственно относящиеся к приложению, которые будут выгружаться на хостинг. Если в момент реализации данной команды проект не будет содержать основной файл index.html, он создастся автоматически, или заменит уже существующий файл, если разработчик подтвердит замену файлов при инициализации приложения.

5. Развернуться приложением.

Для разаертывания приложения на хостинг необходимо вызвать команду развертывания из корневой папки проекта(не из папки приложения). После этого, контент, находящийся в корневой папки приложения будет выгружен на хостинг вашего аккаунта по адресу: идентификатор-проекта-на-хостинге.firebaseapp.com 

> firebase deploy

Внимание! Firebase hosting работает только по безопасному криптографическому протоколу SSL. Это значит, что содержимое обслуживается только по HTTPS. В этой связи необходимо гарантировать, что все сторонние ресурсы, использующиеся приложением и не расположенные на Firebase hosting, загружаются по SSL(HTTPS), включая любые сторонние скрипты. Большинство браузеров не позволяют пользователям подгружать смешанный(HTTP, HTTPS) контент.

Управление развертыванием и откаты.

Находясь на странице консоли (в личном кабинете аккаунта), можно отследить полную историю развертываний приложения, и если разработчик имеет мультихостовый аккаунт, нажав на карточку нужного сайта, может отследить историю его развертываний. Для отката на желаемую версию развертывания, нужно в списке развертываний(История выпусков сайта) выбрать иконку открытия дополнительного меню нужного развертывания, поместив курсор мыши над ним, и нажать на элемент списка Выполнить откат, и приложение откатится на требуемую версию.
