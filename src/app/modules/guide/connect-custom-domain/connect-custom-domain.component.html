Соединение со сторонним доменом

Firebase hosting предоставляет возможность работы хостинга для обслуживания стороннего домена, принадлежащего разработчику, обеспечивая его SSL сертификатом и обслуживания его контента через собственную глобальную сеть доставки контента (CDN). Для этого необходимо совершить несколько шагов.

Настройка связи стороннего домена

Для начала, необходимо создать свой аккаунт и проект на Firebase hosting, и создать приложение (сайт) в этом проекте.

  1. На странице приложения, на закладке Hosting, в консоли проекта Firebase, запустить мастера соединение доменов
    • Если необходимо связать только один домен, нажав на кнопку Подключить домен
    • Если необходимо подключить несколько доменов, выбрать желаемый домен из выпадающего списка, и нажать Подключить домен (доступно для платных планов).
  2. Всести имя нужного домена, намеченного для связи с Firebase Hosting. При желании, можно выбрать настройку перенапрвления всех запросов стороннего домена на иной определяемый домен.
  3. Нажать кнопку Продолжить для запуска процесса проверки. Затем, по необходимости, подтвердить владение доменом, следую инструкциям на экране.
  4. Для завершения настройки связи со сторонним доменом, в большинстве случаев, используется режим Быстрая настройка. Однако, если у вас уже есть работающий сайт у другого провайдера, и необходима миграция без задержек, выберите режим Расширенная настройка из выпадающего списка выбора режима.
    • Quick Setup (быстрая настройка) - укажите динамические пространства имен типа A вашего домена для Firebase hosting.
    • Advanced Setup (расширенная настройка) - следуйте инструкциям на экране, либо обновив записи динамических пространст имен типа TXT стороннего домена, либо выгрузить файлы в специальную область вашего работающего сайта. После завершения проверки можно указать пространства имен типа A к Firebase hosting.
  5. После того, как будут указаны пространства имен типа A к Firebase hosting, в течении 24 часов Firebase обеспечит ваш сторонний домен SSL сертификатом.
Примечание. Если DNS стороннего домена содержит записи типа A или записи типа CNAME, которые указывают на другого провайдера, Firebase не сможет обеспечить SSL сертификат.

Ожидание предоставления SSL сертификата

После подтверждения принадлежности домена, предоставляется SSL сертификат и домен разворачивается через глобальную сеть доставки контента (CDN). Этот процесс занимает некоторое время. Ваш домен будет помещен в список Subject Alternative Names (SAN), который открыт для публичного просмотра. При движении процедуры предоставления сертификата, его можно увидеть как недействительный сертификат не включающий в себя имя вашего домена. Это нормальное поведение при процессе предоставления сертификата, и измениться после завершения процедуры.

Примечание. Firebase Hosting автоматически воспроизводит SSL сертификаты при необходимости, для всех подключенных доменов.
